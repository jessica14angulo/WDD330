Ch14: HTML5 APIs

The data- attribute is a way of embedding data in a web page using custom attributes that are ignored by the browser. The names of these attributes can be decided by the developer, but they must use the following format:

  • Start with data-.
  • Contain only lowercase letters, numbers, hyphens, dots, colons or underscores.
  • Include an optional string value.

data-powers = 'flight superSpeed'

data-rating = '5'

data-dropdown

data-user = 'DAZ'

data-max-length = '32'

Multimedia

An audio clip can be inserted into a page with the audio tag, using the src attribute to point to the audio file. A video clip can be inserted with the video tag, using the src attribute to point to the movie file. The controls attribute can be added (without any value) and will display the browser’s native controls, such as play, pause, and volume control, as can be seen in the screenshot below.

  • The play event, which fires when the clip starts and when it resumes after a pause
  • The pause event, which fires when the clip is paused.
  • The volumechange event, which fires when the volume is changed.
  • The loadedmetadata event, which we saw in the note above, and which fires when all the video's metadata has loaded.

An Introduction to Functions, Execution Context and the Call Stack

  • When we run our code two main things happen: It stores memory and it executes words with a meaning in an application.
  • Execution context is the fact of going through each line of code, executing what they say and store them in memory.
  • There is global memory where every global variable is store and function level or local memory.
  • JavaScript is a language that is single threatened (goes line by line) and uses synchronous execution.
  • When an output of a function is not yet defined its default value is undefined. Its wait to be execute and then stores the new value there.
  • When we call a function, the result becomes a global variable. Using return keyword.
  • Call Stack is tracking our execution context. It uses some kind of layers to keep track of the program.

How to understand Callbacks & Higher Order Functions

  • The principle of making our functions reusable is to change them or generalize them in order to achieve many tasks with a single function but not overloading it.
  • A function declaration is not executed until it is called or stored in a variable.
  • In JavaScript functions are first-class objects. That means that they have all of the properties of objects. We can pass functions that are just objects into functions. They can be stored in variables; they can be stored as properties of other objects or they can be passed as argument’s inputs into functions and they can be returned as the return value.
  • Functions have one special property that objects don’t have, it is that they can be invoked. But functions, despite being objects, can be invoked.
  • Higher-order function is the outer function that takes in a function, and callback function is the function we pass in.
  • High-order and callback function simplify our code, they keep it DRY so we don’t repeat more code. And they let us run asynchronous code.
